🤗 Hugging Face Tokenizers

Complete notes — Pipeline · Components · Code Examples · Methods

Section 1

Why Not Just Split on Spaces?

Whitespace tokenization seems simple but breaks down quickly in practice.

Problems with word-level

"enjoy" and "enjoyed" become separate tokens even though they share a root. New words (scientific terms, names) are never seen during training → OOV problem. Vocabulary can reach millions of entries.

Problems with char-level

Each character is a token. "hello" → [h, e, l, l, o] = 5 tokens. Sequences become extremely long. The model has to learn meaning from scratch — no notion of words.

Sweet spot → Subword tokenization
Vocabulary of 30K–50K tokens. Unknown words split into known subwords. Works across languages including Japanese/Chinese (no spaces). This is what BPE, WordPiece, and SentencePiece all do.
The Hugging Face tokenizers library provides a ready-made, fast implementation of all these algorithms in a single unified API — no need to implement from scratch.
Section 2

The Complete Tokenization Pipeline

Every time you call encode(), the input passes through all 5 stages automatically.

1

Raw Input Text

Whatever the user provides — could be messy, mixed case, with punctuation.

input → "Hmm,, I know I DON'T know"
2

Normalization

Clean and standardize text: lowercase, remove accents, unicode fixes. Makes sure "Café" and "cafe" are treated the same.

lowercase → "hmm,, i know i don't know" strip accents → "cafe" (if café was input)
3

Pre-tokenization

Split text into initial chunks before the main algorithm runs. Common strategy: split on whitespace and punctuation.

whitespace split → ["hmm", ",,", "i", "know", "i", "don't", "know"] whitespace + punct → ["hmm", ",", ",", "i", "know", "i", "don", "'", "t", "know"]
4

Tokenization (Core Algorithm)

Apply the chosen subword algorithm (BPE / WordPiece / SentencePiece) on each pre-token chunk using the learned vocabulary.

BPE example: "playing" → play + ing "don't" → don + 't
5

Post-processing

Add special tokens required by the model. BERT needs [CLS] at start and [SEP] at end. GPT needs <BOS>/<EOS>.

BERT style → [CLS] hmm , , i know i don't know [SEP] GPT style → <BOS> hmm , , i know i don't know <EOS>
Section 3

Components — What Each Stage Does

Normalizer options

Normalizer What it does Example
Lowercase Converts all text to lowercase "Hello" → "hello"
StripAccents Removes accent marks "café" → "cafe"
NFD / NFC / NFKC Unicode normalization forms Fixes encoding inconsistencies
BertNormalizer Lowercase + strip accents + clean text Used in BERT
Sequence Chain multiple normalizers together Lowercase → StripAccents → NFD

Pre-tokenizer options

Pre-tokenizer What it does Example
Whitespace Splits on any whitespace "I love AI" → ["I","love","AI"]
WhitespaceSplit Same but keeps punctuation attached "don't" stays as "don't"
Punctuation Also splits on punctuation chars "don't" → ["don", "'", "t"]
BertPreTokenizer Whitespace + punctuation (BERT standard) Most common for BERT-style models
ByteLevel Splits at byte boundaries (GPT-2 style) Works on raw bytes, language-agnostic

Algorithm options

Algorithm Direction Used in
BPE Merges frequent pairs, vocab grows GPT-2, GPT-3, RoBERTa
WordPiece Merges by score, vocab grows BERT, DistilBERT
Unigram Probabilistic, vocab shrinks T5, ALBERT, XLNet
Section 4

Code — Building a BPE Tokenizer from Scratch

Install first: pip install tokenizers

Step 1 — Import and create the tokenizer

python · setup.py
from tokenizers import Tokenizer from tokenizers.models import BPE from tokenizers.normalizers import Lowercase, Sequence, StripAccents from tokenizers.pre_tokenizers import Whitespace from tokenizers.trainers import BpeTrainer from tokenizers.processors import TemplateProcessing # Step 1: Create tokenizer with BPE algorithm tokenizer = Tokenizer(BPE(unk_token="[UNK]")) # Step 2: Set normalizer (chain lowercase + strip accents) tokenizer.normalizer = Sequence([Lowercase(), StripAccents()]) # Step 3: Set pre-tokenizer (split on whitespace) tokenizer.pre_tokenizer = Whitespace()

Step 2 — Train the tokenizer

python · train.py
# Define the trainer — controls vocabulary size and special tokens trainer = BpeTrainer( vocab_size=30000, min_frequency=2, # token pair must appear at least 2x special_tokens=["[UNK]", "[CLS]", "[SEP]", "[PAD]", "[MASK]"] ) # Provide text files to train on files = ["data/train.txt", "data/valid.txt"] tokenizer.train(files, trainer) # What the tokenizer learned from the corpus: # - vocabulary of 30K subwords # - BPE merge rules (which pairs to combine) # - token frequencies

Step 3 — Add post-processing (BERT style)

python · postprocess.py
# Automatically add [CLS] and [SEP] to every encoded sentence tokenizer.post_processor = TemplateProcessing( single="[CLS] $A [SEP]", pair="[CLS] $A [SEP] $B:1 [SEP]:1", special_tokens=[ ("[CLS]", tokenizer.token_to_id("[CLS]")), ("[SEP]", tokenizer.token_to_id("[SEP]")), ] ) # $A = first sentence, $B = second sentence (for pair tasks like NLI)

Step 4 — Save and reload

python · save_load.py
# Save the trained tokenizer to disk tokenizer.save("my_tokenizer.json") # Reload later (no need to retrain) tokenizer = Tokenizer.from_file("my_tokenizer.json")
Section 5

Key Methods with Examples

tokenizer.encode(text) single sentence

Converts one sentence through the full pipeline → returns an Encoding object.

python
encoding = tokenizer.encode("I love Natural Language Processing") print(encoding.tokens) # ['[CLS]', 'i', 'love', 'natural', 'language', 'processing', '[SEP]'] print(encoding.ids) # [101, 1045, 2293, 3019, 2653, 6364, 102] print(encoding.attention_mask) # [1, 1, 1, 1, 1, 1, 1] ← 1 = real token, 0 = padding
tokenizer.encode_batch(texts) multiple sentences

Encodes a list of sentences simultaneously — much faster than calling encode() in a loop.

python
sentences = [ "Hello world", "I love machine learning", "How are you?" ] encodings = tokenizer.encode_batch(sentences) for enc in encodings: print(enc.tokens) # ['[CLS]', 'hello', 'world', '[SEP]'] # ['[CLS]', 'i', 'love', 'machine', 'learning', '[SEP]'] # ['[CLS]', 'how', 'are', 'you', '?', '[SEP]']
tokenizer.decode(ids) ids → text

Converts token IDs back to a readable string. Useful when the model outputs IDs and you need text.

python
ids = [101, 1045, 2293, 3019, 102] text = tokenizer.decode(ids) print(text) # "i love natural" # Decode batch of sequences at once tokenizer.decode_batch([[101, 1045, 102], [101, 2293, 102]]) # ["i", "love"]
tokenizer.add_special_tokens() / add_tokens() vocabulary extension

Add custom tokens to the vocabulary after training. Special tokens (like [MASK]) are never split by the algorithm.

python
from tokenizers import AddedToken # Add special tokens — treated as single units, never split tokenizer.add_special_tokens(["[MASK]", "[PAD]"]) # Add domain-specific regular tokens tokenizer.add_tokens(["<URL>", "<EMAIL>", "<CODE>"]) # Check the ID of any token tokenizer.token_to_id("[MASK]") # → e.g. 103 tokenizer.id_to_token(103) # → "[MASK]"
Section 6

The Encoding Object — What encode() Returns

encode() doesn't just return IDs. It returns a rich object with everything the model needs.

python · encoding_object.py
enc = tokenizer.encode("I love NLP") enc.tokens # ['[CLS]', 'i', 'love', 'nl', '##p', '[SEP]'] enc.ids # [101, 1045, 2293, 17953, 2361, 102] enc.attention_mask # [1, 1, 1, 1, 1, 1] ← all real tokens enc.type_ids # [0, 0, 0, 0, 0, 0] ← sentence A=0, B=1 for pairs enc.offsets # [(0,0),(0,1),(2,6),(7,9),(9,10),(0,0)] ← char positions enc.special_tokens_mask # [1, 0, 0, 0, 0, 1] ← 1=special, 0=normal

ids

Integer IDs fed directly into the model's embedding layer. Each ID maps to a row in the embedding matrix.

attention_mask

1 = real token, 0 = padding. Used by the transformer to ignore padded positions in batches of different-length sequences.

offsets

Character-level (start, end) position of each token in the original string. Essential for NER and span extraction tasks.

type_ids

0 for sentence A, 1 for sentence B. Used in BERT tasks like NLI or QA where two sentences are paired.

Section 7

Building a WordPiece Tokenizer (BERT Style)

python · bert_tokenizer.py
from tokenizers import Tokenizer from tokenizers.models import WordPiece from tokenizers.normalizers import BertNormalizer from tokenizers.pre_tokenizers import BertPreTokenizer from tokenizers.trainers import WordPieceTrainer from tokenizers.processors import BertProcessing # 1. Create tokenizer with WordPiece algorithm tokenizer = Tokenizer(WordPiece(unk_token="[UNK]")) # 2. Normalize: lowercase + remove accents (BERT standard) tokenizer.normalizer = BertNormalizer() # 3. Pre-tokenize: whitespace + punctuation split tokenizer.pre_tokenizer = BertPreTokenizer() # 4. Train trainer = WordPieceTrainer( vocab_size=30522, # BERT's exact vocab size special_tokens=["[UNK]", "[CLS]", "[SEP]", "[PAD]", "[MASK]"] ) tokenizer.train(["data.txt"], trainer) # 5. Post-process: auto-add [CLS] and [SEP] tokenizer.post_processor = BertProcessing( ("[SEP]", tokenizer.token_to_id("[SEP]")), ("[CLS]", tokenizer.token_to_id("[CLS]")), ) # 6. Encode enc = tokenizer.encode("playing football is fun") print(enc.tokens) # ['[CLS]', 'playing', 'football', 'is', 'fun', '[SEP]'] # WordPiece marks continuation subwords with ## enc2 = tokenizer.encode("unbelievable") print(enc2.tokens) # ['[CLS]', 'un', '##believable', '[SEP]']
The ## prefix is WordPiece's way of marking that a subword is a continuation of the previous one — not the start of a new word. So un + ##believable = "unbelievable".
Section 8

Summary — Everything at a Glance

Stage Class What it does
Normalization Lowercase, BertNormalizer, StripAccents Clean and standardize raw text
Pre-tokenization Whitespace, BertPreTokenizer, ByteLevel Initial split before the algorithm
Algorithm BPE, WordPiece, Unigram Learn and apply the subword vocabulary
Post-processing BertProcessing, TemplateProcessing Add special tokens [CLS], [SEP], etc.
Method Input Output
train(files, trainer) Text files Learned vocabulary
encode(text) One string Encoding object (ids, tokens, mask…)
encode_batch(texts) List of strings List of Encoding objects
decode(ids) List of ints String
save(path) File path JSON file on disk
from_file(path) JSON file path Loaded Tokenizer
add_special_tokens() List of strings Extended vocabulary
Key takeaway: Configure once (normalizer → pre-tokenizer → algorithm → post-processor), train once, then just call encode() — the full pipeline runs automatically every time.
